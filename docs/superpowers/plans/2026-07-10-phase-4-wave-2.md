### Task 2: World-data format-template hardening

**Interfaces produced:**
```cpp
// safe_template.h — validate a printf-style template (from world/builder data) against
// an expected conversion signature before expanding, so a malformed template can't
// read garbage off the stack. On signature MATCH, expands identically to sprintf/
// snprintf today; on MISMATCH, returns the fallback and logs one syslog line.
namespace safe_template
{
    // Conversion signature: the ordered list of specifiers the template MUST contain,
    // exactly (e.g. {S, S} == exactly two %s, no others). Only %s is needed for the
    // current sites; extend the enum only when a real site needs it.
    enum class Conv { String };

    // Expands `tmpl` with `args` into a std::string IFF tmpl's conversions == `expected`
    // (same count, same order, no extras, no %n). Otherwise returns `fallback` (a plain
    // string, no expansion) and calls mudlog once with `context`.
    std::string expand_checked(const char* tmpl,
                               std::initializer_list<Conv> expected,
                               std::initializer_list<std::string_view> args,
                               std::string_view fallback,
                               std::string_view context);
}
```
(Implementer may refine internals; the "validate-then-expand-or-fallback, byte-identical for well-formed" contract is binding.)

- [ ] **Step 1 — RED: signature-match matrix.** In `safe_template_tests.cpp`, test: exactly-2-%s template expands byte-identically to snprintf; too-many-%s → fallback+no-UB; too-few → fallback; wrong-type (`%d` where `%s` expected) → fallback; embedded `%n` → fallback; literal `%%` handled (not counted as a conversion). Run → FAIL (helper absent).
