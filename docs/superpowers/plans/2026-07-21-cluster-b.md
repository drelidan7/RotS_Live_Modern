# Cluster B Wave Implementation Plan (`script.cpp` + the six `shape*.cpp` OLC editors)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Promote the world-growth census's 7-TU Cluster B out of the app tier: the six OLC
editors (`shapemdl.cpp`, `shapemob.cpp`, `shapeobj.cpp`, `shaperom.cpp`, `shapescript.cpp`,
`shapezon.cpp`) into a **NEW `rots_olc` static library** at the top of the L4 band, and
`script.cpp` (the runtime script driver) into **`rots_script`** — the default lean, contingent on
T0's body-read of the `get_param_text` mutual-edge breaker (fallback: `script.cpp` rides into
`rots_olc`). Certified layer order extends to `platform < core < entity < persist < world <
combat < pathfind < script < olc < app`, **every link unidirectional, no bidirectional peer links
ever.** A ninth CI-enforced `OlcLayerAcyclicity` linkcheck lands with the membership.

**Architecture:** Census + closure first (Task 0, read-only), then seams/relocations landed
consumer-free while all 7 TUs stay app-compiled (Task 1), then per-TU call-site conversions
(Tasks 2-3), then the memberships + the new library target + linkcheck as gateable commits
(Task 4), then docs + the standing sweep backlog (Task 5a) and finalization (Task 5b). The wave's
defining constraint is the **`script.cpp ↔ shapescript.cpp` mutual edge** (`script.cpp:347` →
`get_param_text`; `shapescript.cpp:163` → `find_script_by_number`, defined script.cpp:87): under
the no-bidirectional-links invariant the pair cannot split across two libraries unless
`get_param_text` (declared shapescript.cpp:70, heavily used inside shapescript itself) relocates
to `script.cpp` — making shapescript's own uses legal downward calls. Spec:
`docs/superpowers/specs/2026-07-21-cluster-b-olc-wave-design.md`. Recipe:
`docs/superpowers/combat-migration-playbook.md`. Census (scoping): `.superpowers/sdd/world-growth-census.md`
§§4-6. Branch: `arch/cluster-b` off master @`12f9f2d`.

**Tech Stack:** C++20, CMake presets + flat Makefiles, GoogleTest, `nm`-based linkchecks, Python 3
byte-edits for CRLF/formatter-hook files, the existing seam headers (`output_seam.h`,
`combat_hooks.h`, `entity_hooks.h`, `persist_hooks.h`, `world_hooks.h`, `script_hooks.h`) — this
wave adds cells/forwarders to `combat_hooks.h`, `script_hooks.h`, and `output_seam.h`, **one new
seam entry family** (the shared `string_add_init` editor hook), and **one new library target +
linkcheck** (`rots_olc` / `OlcLayerAcyclicity`).

## Global Constraints

- Every moved function body is **byte-verbatim** (reviewers re-diff moved spans). Check each
  file's line-ending profile BEFORE editing; use Python byte-edits where the formatter hook or
  mixed line endings threaten bytes (`rots-formatter-hook-conflict` memory). **Verify the CRLF
  profile of every edited file per file — the shape* family and `script.cpp`/`modify.cpp` have no
  entry in the l4/bw CRLF maps yet; do not assume LF.**
- Call-site conversions and relocations are the ONLY semantic edits allowed, each covered by a
  discriminator or a byte-verbatim re-diff.
- **Layer order becomes `platform < core < entity < persist < world < combat < pathfind < script
  < olc < app`; every link unidirectional, NO bidirectional peer links, ever.** `rots_olc`
  PUBLIC-links `RotS::script` (for `mudlle_converter` + `find_script_by_number`/`get_param_text`
  once script.cpp is in-lib) and transitively the full L3-and-below set. `rots_script` may NEVER
  link `RotS::olc`.
- **Intra-subset closure rule (playbook):** a TU may promote only when closed over every upward
  edge — each such symbol must be (a) owned by a TU promoting in the SAME commit, (b) already
  resolved by a seam/relocation, or (c) already in a library the promoting TU links downward. The
  six shape* editors co-migrate in ONE membership commit (`shapemob` hub fan-in/fan-out makes
  standalone promotion impossible). A census "non-blocking" label is a strong prior, NOT a
  build-wiring guarantee; the `*LayerAcyclicity` linkchecks are ground truth.
- Per-task gates (both hosts): macOS arm64 preset build + ctest, `rots64` container build +
  ctest, both boot goldens (`scripts/boot-golden.sh`), **all existing eight** `*LayerAcyclicity`
  linkchecks green (nine once T4 lands `OlcLayerAcyclicity`), `ConvertEquivalence` 17/17,
  `python3 tools/string_view_census.py --check` exit 0. New/substantially-rewritten test files
  additionally run the `macos-arm64-asan` preset (additive-only is NOT an exemption — trio T2
  lesson). i386 battery at finalization only (Task 5b).
- Characterization goldens (`CharacterizationCombatTest.*`, `CharacterizationJson.*`,
  `PoisonRemovalScriptTest.*`, `boot-golden.sh verify`) must pass **unchanged** — drift = bug.
  `script.cpp` runs zone-reset-triggered scripts during boot, so the boot golden is a real net.
- No death tests (standing rule): abort-tripwire defaults are tested via the registered path +
  the dispatch-flag pattern, never by expecting a crash. No direct `rand()`/`random()` — all
  randomness via `rots_rng`. `std::format` for composition; `-Wall -Wextra -Werror` clean;
  `-funsigned-char` semantics; fixed-size `char[N]` members decay via `static_cast<const char*>`
  before `std::format`.
- STOP-and-adjudicate contract: any upward edge, cascade, or census miss a task surfaces that its
  brief did not disposition → STOP, document evidence, wait for controller ruling. **RIDER GATE
  (carried forward, controller-tracked):** the `script_hooks.h` spec-proc-dispatcher abort-cell
  pattern is pre-authorized for up to THREE total same-shape edges; `virt_program_number`
  (behavior wave) was #1 and `virt_assignmob` is #2 — ONE slot remains. A fourth same-shape edge,
  or ANY different-shape edge, is an auto-STOP.
- Docker gates run SYNCHRONOUSLY in implementer sessions (background container gates stall
  subagents — `subagent-docker-gate-stalls` memory).
- Test-count baseline for this wave: **1446** (behavior wave as-built, both hosts; macOS 75
  skips / rots64 77). Deltas tracked per task, reconciled in Task 5a.
- Implementation subagents run on Sonnet; the T0 census and heavy per-task reviews on Opus; the
  whole-branch review on Fable (model-escalation gate). Briefs generated per task from this plan
  + T0's census; briefs/reports/census live in `.superpowers/sdd/` (gitignored — NEVER commit).
- **Merge-when-green is granted for THIS wave only** (owner, 2026-07-21): battery green +
  whole-branch review clean + all six blocking CI jobs green on the PR → controller ff-merges to
  master and deletes the branch. No standing authority carries forward.
