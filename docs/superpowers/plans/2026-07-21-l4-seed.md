# L4-Seed Wave Implementation Plan (rots_pathfind + rots_script stand up an L4 band; zone.cpp → rots_world)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use
> checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the **first two libraries of a new L4 orchestration/authoring band** above the L3
peer tier — `rots_pathfind` (`graph.cpp`) and `rots_script` (`mudlle.cpp` + `mudlle2.cpp`) — with two
new linkchecks (`PathfindLayerAcyclicity`, `ScriptLayerAcyclicity`) mirroring the six existing ones,
and ride `zone.cpp`'s reset half into `rots_world` (L3) independently. Certified layer order:
`platform < core < entity < persist < world < combat < pathfind < script < app`, **every link
unidirectional, no bidirectional peer links ever.**

**Architecture:** Census + closure first (Task 0, read-only), then seams/relocations/retirements
landed consumer-free while all TUs stay app-compiled (Task 1), then per-TU call-site conversions
(Task 2), then the membership moves + the two new CMake library targets (Task 3), then docs (Task 4),
then finalization (Task 5). The owner's L4-band placement dissolves the world-growth census's §7
bidirectional-link finding for graph/mudlle (their `hit()`/`CAN_SEE()`/`update_pos()` calls become
legal **downward** L4→L3 edges); the combat/app coupling instead lands on `zone.cpp`, whose edges
into `rots_combat`/app tiers are inverted via `world_hooks.h` hooks or relocation (the
no-bidirectional rule forbids the `world → combat` link that would otherwise resolve them). Spec:
`docs/superpowers/specs/2026-07-21-l4-seed-design.md`. Recipe:
`docs/superpowers/combat-migration-playbook.md`. Census:
`.superpowers/sdd/world-growth-census.md`. Branch: `arch/l4-seed` off master @b6f6b76.

**Tech Stack:** C++20, CMake presets + flat Makefiles, GoogleTest, `nm`-based linkchecks, Python 3
(byte-edits for CRLF/formatter-hook files), the existing seam headers (`output_seam.h`,
`combat_hooks.h`, `entity_hooks.h`, `persist_hooks.h`, `world_hooks.h`) plus one new header
(`script_hooks.h`).

## Global Constraints

- Every moved function body is **byte-verbatim** (reviewers re-diff moved spans). CRLF/LF: check each
  file's line-ending profile BEFORE editing; use Python byte-edits where the formatter hook or mixed
  line endings threaten bytes (established method — `rots-formatter-hook-conflict` memory). All four
  candidate files (`graph.cpp`/`mudlle.cpp`/`mudlle2.cpp`/`zone.cpp`) read as plain LF at @b6f6b76 —
  **verify per file anyway, do not assume** (the combat-pilot wave hit CRLF in `clerics.cpp`, LF in
  `fight.cpp`).
- Call-site conversions are the ONLY semantic edits allowed, each covered by a discriminator test.
- **Layer order is `platform < core < entity < persist < world < combat < pathfind < script < app`;
  every link is unidirectional and there are NO bidirectional peer links, ever.** `rots_combat`
  already PUBLIC-links `RotS::world`, so `rots_world` may never link `RotS::combat` — any zone edge
  into combat must be inverted (hook) or relocated downward, never link-resolved. `rots_script`
  PUBLIC-links `RotS::pathfind` (the one sanctioned intra-band edge); `rots_pathfind` never
  references a `rots_script` symbol.
- **Intra-subset closure rule (playbook, "The intra-subset rule" + "Census-methodology
  correction"):** a TU may promote only when it is closed over every upward edge — each such symbol
  must be (a) owned by a TU promoting in the SAME commit, (b) already resolved by a seam/relocation,
  or (c) already in a library the promoting TU links downward. A census's "non-blocking" label is a
  strong prior, NOT a build-wiring guarantee; the `*LayerAcyclicity` linkcheck is ground truth. Run
  the closure check BEFORE picking the membership set, not after a build failure. Cluster A
  `{graph, mudlle, mudlle2}` is self-closed and `zone` is isolated (census §6), but the intra-band
  `script → pathfind` edge means `rots_pathfind` must exist before `rots_script`'s linkcheck runs.
- Per-task gates (both hosts): macOS arm64 preset build + ctest, `rots64` container build + ctest,
  both boot goldens (`scripts/boot-golden.sh`), **all eight** `*LayerAcyclicity` linkchecks green
  (the six existing + the two new), `ConvertEquivalence` 17/17, `python3 tools/string_view_census.py
  --check` exit 0. New or substantially rewritten test files additionally run the `macos-arm64-asan`
  preset. i386 battery at finalization only (Task 5).
- Characterization goldens (`CharacterizationCombatTest.*`, `CharacterizationJson.*`,
  `PoisonRemovalScriptTest.*`, `boot-golden.sh verify`) must pass **unchanged** — drift = bug in the
  change. `mudlle`/`graph`/`zone` are boot-golden-exercised (mob programs, hunting, zone resets), so
  the boot golden is a real net here, not a formality.
- No death tests (standing rule). No direct `rand()`/`random()` — all randomness via `rots_rng`.
  `std::format` for composition; `-Wall -Wextra -Werror` clean; `-funsigned-char` semantics.
- STOP-and-adjudicate contract: any upward edge, cascade, or census miss a task surfaces that its
  brief did not disposition → STOP, document evidence, wait for controller ruling. The most likely
  STOP is `equip_char` body-reading as combat-coupled (not entity-relocatable) or a zone linkcheck
  cascade surfacing an edge T0 missed.
- Docker gates must run SYNCHRONOUSLY in implementer sessions (background container gates stall
  subagents — `subagent-docker-gate-stalls` memory).
- Test-count baseline for this wave: **1398** (combat-trio as-built, both hosts). Deltas tracked per
  task, reconciled in Task 4.
- Implementation subagents run on Sonnet (model-escalation gate); briefs generated per task from this
  plan + T0's census.
